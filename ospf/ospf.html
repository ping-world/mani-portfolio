Project Overview

This is a CCNP-style, multi-area OSPF lab built in GNS3 to demonstrate hierarchical design, LSA control (stub/NSSA), DR/BDR behavior on broadcast segments, route summarization, security, and a virtual-link recovery scenario.

Topology

14 routers + 2 Ethernet switches (GNS3 “Ethernet switch” nodes).

Core (Area 0)
  • R1—R2—R3—R4 in a ring.
  • “CORE-LAN” broadcast segment on SW1 in Area 0.
Branches (multi-area)
  • Area 10 (Totally Stub): R2 (ABR) connects to R5 and R6.
  • Area 20 (NSSA): R3 (ABR) connects to R9 (internal) and R10 (ASBR doing redistribution); NSSA externals are Type-7 inside the NSSA and get translated to Type-5 toward the rest of the domain by the translating ABR.
  • Area 30 (Normal): R4 (ABR) connects to R11 and R12 via SW2 (broadcast segment to practice DR tuning).
  • Area 40 (Normal transit): R4 connects to R7; used as the transit area for the virtual-link scenario.
  • Area 50 (Remote normal): R7 (ABR) connects to R8; restored to Area 0 connectivity via virtual link (a common “fix” when an area isn’t physically attached to Area 0).
External / “ISP”
  • R10 connects to R13 (ISP simulation). Create static routes on R13 and redistribute at R10.

Addressing & Area Plan

  • Loopbacks (Router-IDs + test targets): 10.255.<router#>.1/32 (e.g., R1 Lo0 = 10.255.1.1/32).
  • P2P links: 10.<area>.<link#>.0/31 (or /30 if preferred).
  • Optional LANs behind branches: 10.<area>.<router#>.0/24 (loopback or dummy LAN).
  • Areas: 0 (core + SW1), 10 (totally stub), 20 (NSSA), 30 (normal + SW2), 40 (transit), 50 (remote via virtual-link).

Implementation “Tickets” (What I built + verified)

  1. Baseline build: hostnames, disable DNS lookup, Loopback0 router-id, IP addressing, ping reachability to loopbacks.
  2. Multi-area OSPF foundation: single process, correct area placement, passive-interface default with selective unpassive on neighbor links.
  3. DR/BDR & network types: tune DR/BDR on SW1 and SW2 using OSPF priority; validate with show ip ospf neighbor.
  4. Area 10 totally stub: confirm branch routers learn a default route (instead of full inter-area/external detail), matching stub/totally-stub behavior.
  5. Area 20 NSSA + redistribution: inject ISP routes at R10; prove Type-7 inside NSSA and Type-5 outside via ABR translation.
  6. Summarization: inter-area summarization on ABRs (area <id> range ...) and external summarization on ASBR (summary-address ...).
  7. Security drill: enable OSPF authentication, break one adjacency, document symptoms and fix.
  8. Virtual-link scenario: demonstrate Area 50 broken without Area 0 reachability, then restore via a virtual link through Area 40; verify with show ip ospf virtual-links.
